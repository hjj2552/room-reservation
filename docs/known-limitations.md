# 현재 제한 사항

이 문서는 현재 MVP에서 지원하지 않는 것과 운영상 주의가 필요한 범위를 명확히 남기기 위한 문서입니다. 아래 항목은 숨겨진 TODO가 아니라, 인수자가 기능 범위를 판단할 때 참고해야 하는 현재 상태입니다.

## 현재 MVP에서 지원하지 않는 것

관리자 계정 관리는 아직 화면으로 제공되지 않습니다.

- 관리자 계정 등록
- 관리자 비밀번호 변경
- 관리자 역할/권한 분리
- 계정 잠금, 비활성화, 로그인 실패 횟수 제한
- 관리자별 상세 권한 설정

운영 정책은 전역 설정만 지원합니다.

- 공간별 운영 시간
- 공간별 예약 가능 요일
- 휴일, 임시 휴무일, 시험 기간 예외
- 특정 날짜만 막는 블랙아웃 일정
- 신청자별 예약 한도
- 예약 가능 시작일 또는 사전 예약 가능 기간 제한

예약 기능은 등록/조회/수정/승인/취소/삭제와 단건 복제를 지원합니다.

- 취소 예약 복구
- 삭제 예약 복구
- 대기열
- 부분 승인
- 다중 공간 동시 예약
- 첨부파일
- 장비 대여
- 승인 라인 또는 다단계 결재
- 신청자 본인 확인

반복 예약은 여러 일반 예약을 한 번에 등록하는 묶음 중심 기능입니다.

- 반복 예약 수정
- 반복 예약 규칙 복사
- 반복 예약 충돌 해결 UI

알림과 외부 연동은 아직 없습니다.

- 이메일 발송
- 문자, 카카오톡, Slack 등 알림
- 캘린더 연동
- 학사 시스템 또는 SSO 연동
- 웹훅

감사/이력 기능은 예약 감사 이력 중심입니다.

- 운영 설정 변경 이력
- 공간 변경 이력
- 관리자 로그인/로그아웃 이력
- CSV 내보내기 이력
- 감사 이력 CSV 내보내기

공개 사용자 영역은 `/timetable`의 날짜별·공간별 시간표, 예약 신청, 마스킹 상세 조회, 비밀번호 기반 수정·취소를 지원합니다. 기능 정책은 `docs/public-reservation.md`에 정리되어 있지만, 실제 배포 환경의 사용자 지원·장애 대응 절차는 아직 없습니다.

## 운영상 주의가 필요한 것

관리자 인증은 현재 MVP 수준입니다. 백엔드 설정값의 관리자 계정으로 로그인하며, `admins` 테이블 기반의 계정 관리와 역할 분리는 아직 실제 로그인 흐름에 연결되어 있지 않습니다.

local/dev/prod profile은 `ADMIN_USERNAME`, `ADMIN_PASSWORD` 환경변수가 없으면 시작하지 않습니다. `admin` / `admin1234`는 test/E2E profile의 폐기 가능한 기본값일 뿐 운영 계정으로 사용하면 안 됩니다.

관리자 인증은 서버 세션 쿠키를 사용합니다. 상태 변경 요청은 공개 예약 등록·수정·취소를 포함해 CSRF 검증을 거치며, SPA는 `XSRF-TOKEN` 쿠키 값을 `X-XSRF-TOKEN` 헤더로 전송합니다.

공개·비로그인 API 요청은 인메모리 토큰 버킷으로 IP별 제한됩니다. GET은 분당 120회, 그 밖의 메서드는 분당 24회이며 인증된 `ROLE_ADMIN` 요청은 제외됩니다. 현재 버킷은 서버 인스턴스별 메모리에 있으므로 다중 인스턴스 배포에서는 전역 제한을 보장하지 않습니다.

세션 쿠키는 모든 profile에서 `SameSite=Lax`, `HttpOnly=true`를 사용합니다. common/prod 기본은 `Secure=true`이고 local/E2E의 HTTP 실행만 `Secure=false`로 덮어씁니다.

시간대는 서비스 로직과 CSV 내보내기에서 `Asia/Seoul` 기준으로 처리합니다. 해외 사용자 또는 다른 시간대 운영은 별도 검토가 필요합니다.

운영 설정 변경은 기존 예약을 자동 정리하지 않습니다. 예를 들어 운영 시간을 줄여도 이미 존재하는 예약이 자동 취소되지는 않습니다.

공간 비활성화는 새 예약 등록을 막기 위한 상태입니다. 이미 존재하는 예약을 자동 취소하지는 않습니다.

충돌 검사는 `REQUESTED`, `CONFIRMED` 예약을 모두 점유 중으로 봅니다. 승인 대기 예약도 같은 시간대의 새 예약을 막습니다.

반복 예약 취소는 연결된 활성 예약을 모두 취소합니다. 일부 회차 취소는 등록된 개별 예약 상세에서 처리합니다.

CSV 내보내기에는 신청자 이름, 이메일, 전화번호, 목적이 포함될 수 있습니다. 파일 공유와 보관에 주의해야 합니다.

로컬 개발 DB는 Docker volume을 사용합니다. `docker compose down`만으로 데이터가 지워지지는 않지만, volume 삭제나 DB 초기화 작업은 데이터 손실을 만들 수 있습니다.

테스트 DB인 `postgres-test`는 tmpfs 기반입니다. 컨테이너를 다시 만들면 데이터가 사라지는 전제입니다.

## 향후 확장 후보

관리자 운영 품질을 높이기 위한 후보는 다음과 같습니다.

- 관리자 계정/권한 관리 화면
- 비밀번호 변경과 비밀번호 정책
- 삭제한 공간 또는 예약 복구
- 운영 설정 변경 이력
- 공간 변경 이력
- 공개 화면의 배포·사용자 지원·장애 대응 운영 절차
- 반복 예약 수정, 일부 취소, 연결 예약 필터
- 휴일/예외일 관리
- 신청자별 예약 제한
- 이메일 또는 문자 알림
- 예약 승인/취소 알림 템플릿
- CSV 내보내기 범위 선택과 내보내기 이력
- 개인정보 마스킹과 보관 기간 정책
- 운영 대시보드
- 장애 진단용 헬스체크와 관리자용 상태 페이지

개발 검증 품질을 높이기 위한 후보는 다음과 같습니다.

- 프런트엔드 단위 테스트
- API contract 테스트
- CSV 내보내기 E2E
- 관리자 화면 전체 CRUD matrix E2E
- 접근성 테스트
- 시각 회귀 테스트
- 운영 설정 변경에 따른 기존 예약 영향 점검 테스트

## 테스트는 있지만 아직 UI로 완전히 검증하지 않은 부분

백엔드 통합 테스트는 다음 영역을 다룹니다.

- 공개 예약 등록, 충돌, 정책 검증
- 관리자 예약 목록/상세/승인/취소
- 관리자 예약 등록/수정
- 반복 예약 미리보기/등록/취소
- 예약 감사 이력
- CSV 내보내기
- 공간 API
- 운영 설정 API
- 관리자 인증 API
- CSRF 보호
- IP별 Rate Limiting과 관리자 우회
- 태그 CRUD와 반복 예약 참조 해제
- E2E 테스트 데이터 cleanup 보호 조건

관리자 Playwright E2E는 핵심 smoke 흐름을 다룹니다.

- 인증과 보호 라우트
- 예약 목록 검색 조건 유지
- 감사 이력 검색 조건 유지
- 관리자 예약 등록과 목록/상세 반영
- 관리자 예약 수정과 목록/상세 반영
- 관리자 시간표의 날짜별·공간별 보기와 빈 슬롯 예약 등록
- 예약 상세의 시간표 이동, 단건 복제, 영구 삭제와 삭제 감사 이력
- 공간 목록·수정·영구 삭제와 예약 기록 보존 안내
- 운영 설정 로드, 저장, 성공 피드백
- 반복 예약 미리보기·등록·상세·취소와 `SKIP_CONFLICTS`
- 반복 예약 태그 선택과 태그 생성·수정·삭제
- 공개 시간표 예약 신청, 마스킹 상세, 비밀번호 취소, 승인 예약 수정 후 승인 대기 전환

아직 UI로 완전히 검증하지 않은 부분은 다음과 같습니다.

- CSV 내보내기 파일 내용 검증
- 공간 등록/수정/활성화/비활성화 전체 조합
- 운영 설정 모든 필드의 경계값 검증
- 반복 예약 모든 충돌 정책 조합
- 반복 예약 취소 후 연결 예약 전체 검증
- 예약 승인/취소 실패 케이스별 UI 메시지
- 공개 예약 비밀번호 오류와 정책 경계값의 전체 UI 조합
- 모바일 또는 작은 화면 레이아웃
- 접근성 키보드 탐색

## 문서상 아직 비어 있는 영역

운영 환경 배포 방식이 확정되지 않아 다음 문서는 아직 상세 절차가 없습니다.

- 운영 서버 URL과 접속 경로
- 운영 관리자 계정 발급/회수 절차
- 운영 DB 백업/복구 절차
- 장애 대응 연락망
- 개인정보 보관/파기 정책
- 실제 학교 또는 기관 운영 정책에 맞춘 문구 예시

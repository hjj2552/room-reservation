# 운영자용 관리자 매뉴얼

이 문서는 후임 운영자가 코드 수정 없이 관리자 화면에서 강의실 예약을 운영하기 위한 초안입니다. 실제 운영 환경의 URL, 계정 발급 방식, 백업 절차는 배포 방식이 확정되면 별도로 보강해야 합니다.

## 시스템 목적과 대상 사용자

강의실 예약 시스템은 학생, 교직원 등 사용자가 강의실 사용을 신청하고, 운영자가 관리자 화면에서 강의실, 운영 정책, 예약 상태, 반복 예약, 감사 이력을 관리하기 위한 MVP입니다.

대상 사용자는 다음과 같습니다.

- 예약 운영자: 예약 승인, 수정, 취소, 반복 예약 등록, 이력 확인을 담당합니다.
- 시설 또는 학사 담당자: 강의실 목록과 운영 기간, 운영 시간, 예약 가능 요일을 관리합니다.
- 개발/운영 인수자: 장애 발생 시 현재 구현 범위와 제한 사항을 확인합니다.

## 로그인 방법

1. 관리자 프런트엔드 주소로 접속합니다. 로컬 개발 환경에서는 보통 `http://localhost:5173`입니다.
2. 로그인 화면에서 관리자 아이디와 비밀번호를 입력합니다.
3. 로컬 기본 계정은 `admin` / `admin1234`입니다.
4. 로그인이 성공하면 예약 목록 화면으로 이동합니다.
5. 작업을 마치면 왼쪽 메뉴 하단의 로그아웃 버튼으로 세션을 종료합니다.

주의: 현재 MVP의 관리자 인증은 애플리케이션 설정값 기반의 단일 운영자 계정에 가깝습니다. 운영 환경에서는 기본 비밀번호를 그대로 쓰면 안 되며, 환경 변수 또는 운영 설정 파일로 별도 주입해야 합니다.

## 강의실 관리 방법

왼쪽 메뉴에서 `강의실 관리`로 이동합니다.

강의실 목록에서 할 수 있는 작업은 다음과 같습니다.

- 강의실명 또는 위치 키워드로 검색합니다.
- 강의실의 운영 상태, 이름, 위치, 정원, 마지막 수정 시각을 확인합니다.
- `수정` 버튼으로 기존 강의실 정보를 오른쪽 입력 폼에 불러옵니다.
- `비활성화` 또는 `활성화` 버튼으로 공개 예약 가능 여부를 바꿉니다.

새 강의실을 등록하려면 다음 순서로 진행합니다.

1. 오른쪽 강의실 입력 폼에서 강의실명, 위치, 정원, 설명을 입력합니다.
2. 예약 가능한 상태로 운영하려면 `예약 가능 상태로 운영`을 체크합니다.
3. `강의실 등록`을 누릅니다.
4. 목록에 새 강의실이 표시되는지 확인합니다.

기존 강의실을 수정하려면 다음 순서로 진행합니다.

1. 목록에서 수정할 강의실의 `수정`을 누릅니다.
2. 오른쪽 폼에서 값을 변경합니다.
3. `수정 저장`을 누릅니다.
4. 목록의 수정 시각과 변경된 값이 갱신되는지 확인합니다.

주의: 백엔드에는 강의실 소프트 삭제 API가 있지만 현재 관리자 화면에는 삭제/복구 버튼이 없습니다. 운영자는 화면에서 활성화/비활성화와 정보 수정만 할 수 있습니다.

## 운영 설정 관리 방법

왼쪽 메뉴에서 `운영 설정`으로 이동합니다.

운영 설정에서 관리하는 항목은 다음과 같습니다.

- 기관명
- 공개 예약 접수 사용 여부
- 공개 안내 문구
- 접수 중지 안내 문구
- 학기 시작일과 종료일
- 운영 시작 시간과 종료 시간
- 예약 단위
- 최소/최대 예약 시간
- 전화번호 필수 입력 여부
- 예약 가능 요일
- 담당자 이름, 이메일, 전화번호
- 예약 완료 안내 문구

설정 변경 절차는 다음과 같습니다.

1. 현재 설정 값을 확인합니다.
2. 필요한 항목만 수정합니다.
3. 예약 가능 기간, 시간, 요일, 예약 단위가 서로 맞는지 확인합니다.
4. `설정 저장`을 누릅니다.
5. 저장 성공 메시지가 보이면 변경이 반영된 상태입니다.

운영상 중요한 점은 다음과 같습니다.

- `공개 예약 접수 사용`을 끄면 공개 예약 등록이 막힙니다. 이미 등록된 예약을 자동 취소하지는 않습니다.
- 학기 기간 밖의 날짜는 예약할 수 없습니다.
- 운영 시간 밖의 시간대는 예약할 수 없습니다.
- 예약 단위는 백엔드 제약상 `10`, `15`, `30`, `60`분 단위만 안전하게 사용해야 합니다.
- 전화번호 필수 입력을 켜면 예약 등록 시 전화번호가 비어 있으면 실패합니다.

## 단건 예약 등록/수정/취소 방법

왼쪽 메뉴에서 `예약 등록` 또는 `예약 목록`으로 이동합니다.

단건 예약 등록 절차는 다음과 같습니다.

1. `예약 등록` 화면을 엽니다.
2. 강의실을 선택합니다.
3. 상태를 선택합니다. 관리자 등록 예약의 기본 상태는 `확정`입니다.
4. 신청자 이름, 이메일, 전화번호, 예약 목적을 입력합니다.
5. 시작 시간과 종료 시간을 입력합니다.
6. 필요한 경우 처리 메모를 남깁니다.
7. 저장합니다.
8. 저장 후 예약 상세 화면에서 입력 내용과 상태를 확인합니다.

예약 수정 절차는 다음과 같습니다.

1. `예약 목록`에서 예약을 검색합니다.
2. 예약 상세 화면으로 들어갑니다.
3. `예약 수정`을 누릅니다.
4. 강의실, 신청자 정보, 목적, 시간, 상태, 메모를 수정합니다.
5. 저장 후 상세 화면에서 변경 내용을 확인합니다.

예약 승인/취소 절차는 다음과 같습니다.

1. 예약 상세 화면으로 이동합니다.
2. 처리 메모에 승인 또는 취소 사유를 적습니다.
3. 승인하려면 `승인 처리`를 누릅니다.
4. 취소하려면 `취소 처리`를 누릅니다.
5. 화면 하단 감사 이력에서 상태 변경과 메모가 남았는지 확인합니다.

상태 의미는 다음과 같습니다.

- `REQUESTED`: 승인 대기 상태입니다.
- `CONFIRMED`: 확정된 예약입니다.
- `CANCELLED`: 취소된 예약입니다.

주의: `REQUESTED`와 `CONFIRMED`는 모두 충돌 검사 대상입니다. 같은 강의실의 시간이 겹치면 새 예약 등록 또는 수정이 실패합니다. `CANCELLED` 예약은 충돌 대상에서 제외됩니다.

## 반복 예약 preview/create/cancel 방법

왼쪽 메뉴에서 `반복 예약`으로 이동합니다.

반복 예약은 먼저 미리보기로 충돌 여부를 확인한 뒤 등록합니다.

반복 예약 미리보기 절차는 다음과 같습니다.

1. 강의실을 선택합니다.
2. 신청자 이름, 이메일, 전화번호, 예약 목적을 입력합니다.
3. 시작일과 종료일을 입력합니다.
4. 시작 시간과 종료 시간을 입력합니다.
5. 반복 요일을 선택합니다.
6. 등록 정책을 선택합니다.
7. `미리보기`를 누릅니다.
8. 전체 후보 수, 가능 수, 충돌 수를 확인합니다.
9. 미리보기 표에서 날짜별 가능/충돌 결과와 사유를 확인합니다.

반복 예약 등록 절차는 다음과 같습니다.

1. 미리보기 결과를 확인합니다.
2. 등록 정책이 의도와 맞는지 다시 확인합니다.
3. `반복 예약 등록`을 누릅니다.
4. 등록 완료 결과의 등록 건수, 건너뛴 건수, 실패 건수를 확인합니다.
5. 반복 예약 목록 또는 예약 목록에서 실제 등록된 예약을 확인합니다.

반복 예약 등록 정책은 다음과 같이 해석합니다.

- `FAIL_ALL`: 충돌 또는 정책 위반이 하나라도 있으면 전체 등록을 중단합니다.
- `SKIP_CONFLICTS`: 충돌 또는 정책 위반 건은 건너뛰고 가능한 건만 등록합니다.
- `CREATE_AVAILABLE_ONLY`: 가능한 건만 등록합니다. 현재 동작은 `SKIP_CONFLICTS`와 유사하게 이해하면 됩니다.

반복 예약 취소 절차는 다음과 같습니다.

1. 반복 예약 목록에서 취소할 항목의 `취소`를 누르거나 상세 화면으로 들어갑니다.
2. 취소 메모를 입력합니다.
3. `반복 예약 취소`를 누릅니다.
4. 반복 예약 상태가 취소로 바뀌었는지 확인합니다.
5. 연결된 활성 예약이 `CANCELLED`로 바뀌었는지 예약 목록에서 확인합니다.

주의: 반복 예약 취소는 반복 예약 자체를 소프트 삭제하고, 연결된 활성 예약을 취소합니다. 이미 취소된 예약은 다시 취소할 필요가 없습니다.

## 충돌이 발생했을 때 해석 방법

충돌은 같은 강의실에서 예약 시간이 겹치는 경우입니다. 시스템은 `REQUESTED`, `CONFIRMED` 상태 예약을 모두 점유 중인 시간으로 봅니다.

자주 보는 충돌/정책 사유는 다음과 같습니다.

- `TIME_SLOT_CONFLICT`: 같은 강의실의 기존 예약과 시간이 겹칩니다.
- `ROOM_DISABLED`: 해당 강의실이 예약 가능 상태가 아닙니다.
- `RESERVATION_DISABLED`: 공개 예약 접수가 중지되어 있습니다.
- `OUTSIDE_SEMESTER_PERIOD`: 운영 설정의 학기 기간 밖입니다.
- `OUTSIDE_OPERATING_DAYS`: 예약 가능 요일이 아닙니다.
- `OUTSIDE_OPERATING_HOURS`: 운영 시간 밖이거나 하루를 넘기는 예약입니다.
- `INVALID_DURATION`: 최소/최대 예약 시간 범위를 벗어났습니다.
- `INVALID_SLOT_UNIT`: 설정된 예약 단위에 맞지 않는 시간입니다.
- `VALIDATION_ERROR`: 필수 입력 누락, 시작/종료 시간 오류 등 입력값 문제입니다.

충돌이 나면 먼저 다음 순서로 확인합니다.

1. 같은 강의실의 같은 날짜 예약 목록을 확인합니다.
2. 취소되지 않은 예약과 시간이 겹치는지 확인합니다.
3. 운영 설정의 기간, 요일, 시간, 예약 단위와 맞는지 확인합니다.
4. 반복 예약이라면 미리보기에서 충돌 날짜만 건너뛸지, 전체 등록을 중단할지 결정합니다.

## CSV 내보내기 사용 방법

예약 목록 화면에서 현재 검색 조건으로 CSV를 내보낼 수 있습니다.

사용 절차는 다음과 같습니다.

1. `예약 목록`으로 이동합니다.
2. 상태, 강의실, 시작일, 종료일, 검색어 조건을 설정합니다.
3. 조회 결과가 의도한 범위인지 확인합니다.
4. `CSV 내보내기`를 누릅니다.
5. `reservations.csv` 파일을 내려받아 엽니다.

CSV에는 예약 ID, 강의실명, 신청자명, 이메일, 전화번호, 목적, 시작/종료 시각, 상태, 등록 경로, 반복 예약 ID, 등록 시각이 포함됩니다. 시간은 한국 시간 기준 `yyyy-MM-dd HH:mm:ss` 형식으로 내려갑니다. Excel 호환을 위해 UTF-8 BOM이 포함됩니다.

주의: 왼쪽 메뉴의 `CSV 내보내기` 링크는 별도 필터 없이 전체 예약 CSV를 요청합니다. 특정 조건만 내려받으려면 예약 목록 화면에서 조건을 적용한 뒤 `CSV 내보내기`를 사용하세요.

## audit/history 확인 방법

예약별 감사 이력은 예약 상세 화면 하단에서 확인합니다.

전체 감사 이력은 왼쪽 메뉴의 `감사 이력`에서 확인합니다.

감사 이력 검색 절차는 다음과 같습니다.

1. `감사 이력`으로 이동합니다.
2. 필요한 경우 예약 ID, 강의실, 처리 유형, 시작일, 종료일을 입력합니다.
3. `조회`를 누릅니다.
4. 처리 시각, 처리 유형, 상태 변경, 처리자, 연결 예약, 메모를 확인합니다.
5. 특정 예약을 자세히 보려면 `상세 보기`를 누릅니다.

주요 처리 유형은 다음과 같습니다.

- `CREATED`: 공개 예약 등록
- `CREATED_BY_ADMIN`: 관리자 예약 등록
- `RECURRENCE_GENERATED`: 반복 예약 건 등록
- `UPDATED`: 예약 수정
- `APPROVED`: 예약 승인
- `CANCELLED`: 예약 취소
- `RECURRENCE_CANCELLED`: 반복 예약 취소로 연결 예약 취소

주의: 현재 감사 이력은 예약 감사 이력 중심입니다. 강의실 변경, 운영 설정 변경, 로그인 이력은 별도 감사 화면으로 제공되지 않습니다.

## 운영 시 주의사항

- 운영 시간, 학기 기간, 예약 가능 요일을 바꾸면 이후 등록/수정되는 예약에 정책이 적용됩니다.
- 이미 존재하는 예약이 운영 설정 변경으로 자동 취소되지는 않습니다.
- 같은 강의실 같은 시간대에는 승인 대기와 확정 예약이 동시에 존재할 수 없습니다.
- 반복 예약을 등록하기 전에는 반드시 미리보기 충돌 수를 확인합니다.
- CSV에는 신청자 개인정보가 포함될 수 있으므로 공유 범위를 제한합니다.
- 로컬 기본 관리자 비밀번호는 운영 환경에서 사용하지 않습니다.
- 운영자가 화면에서 관리자 계정을 추가하거나 비밀번호를 변경할 수 없습니다.
- 장애 조사 시 예약 상세 이력과 감사 이력을 먼저 확인합니다.

## 아직 미구현 또는 운영자가 할 수 없는 것

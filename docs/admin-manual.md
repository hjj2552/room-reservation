# 관리자 매뉴얼

이 문서는 후임 관리자가 코드 수정 없이 관리자 화면에서 강의실 예약을 운영하기 위한 초안입니다. 실제 운영 환경의 URL, 계정 발급 방식, 백업 절차는 배포 방식이 확정되면 별도로 보강해야 합니다.

## 시스템 목적과 대상 사용자

강의실 예약 시스템은 학생, 교직원 등 사용자가 강의실 사용을 신청하고, 관리자가 관리자 화면에서 강의실, 운영 정책, 예약 상태, 반복 예약, 감사 이력을 관리하기 위한 MVP입니다.

대상 사용자는 다음과 같습니다.

- 관리자: 예약 승인, 수정, 취소, 반복 예약 등록, 이력 확인을 관리합니다.
- 관리자: 강의실 목록과 운영 기간, 운영 시간, 예약 가능 요일을 관리합니다.
- 개발/운영 인수자: 장애 발생 시 현재 구현 범위와 제한 사항을 확인합니다.

## 로그인 방법

1. 프런트엔드 주소로 접속합니다. 로컬 개발 환경에서는 보통 `http://localhost:5173`입니다.
2. 로그인 화면에서 관리자 아이디와 비밀번호를 입력합니다.
3. local 환경에서는 저장소 루트 `.env`의 `ADMIN_USERNAME`, `ADMIN_PASSWORD`로 로그인합니다.
4. 로그인이 성공하면 예약 목록 화면으로 이동합니다.
5. 작업을 마치면 왼쪽 메뉴 하단의 로그아웃 버튼으로 세션을 종료합니다.

주의: 현재 MVP의 관리자 인증은 애플리케이션 설정값 기반의 단일 관리자 계정에 가깝습니다. 운영 환경에서는 기본 비밀번호를 그대로 쓰면 안 되며, 환경 변수 또는 운영 설정 파일로 별도 주입해야 합니다.

## 강의실 관리 방법

왼쪽 메뉴에서 `강의실 관리`로 이동합니다.

강의실 목록에서 할 수 있는 작업은 다음과 같습니다.

- 강의실명 또는 위치 키워드로 검색합니다.
- 강의실의 운영 상태, 이름, 위치, 정원, 마지막 수정 시각을 확인합니다.
- `수정` 버튼으로 기존 강의실 정보를 오른쪽 입력 폼에 불러옵니다.
- `비활성화` 또는 `활성화` 버튼으로 공개 예약 가능 여부를 바꿉니다.

새 강의실을 등록하려면 다음 순서로 진행합니다.

1. 오른쪽 강의실 입력 폼에서 강의실명, 위치, 정원, 설명을 입력합니다.
2. 예약 가능한 상태로 운영하려면 `예약 가능 상태로 운영`을 체크합니다.
3. `강의실 등록`을 누릅니다.
4. 목록에 새 강의실이 표시되는지 확인합니다.

기존 강의실을 수정하려면 다음 순서로 진행합니다.

1. 목록에서 수정할 강의실의 `수정`을 누릅니다.
2. 오른쪽 폼에서 값을 변경합니다.
3. `수정 저장`을 누릅니다.
4. 목록의 수정 시각과 변경된 값이 갱신되는지 확인합니다.

강의실을 영구 삭제하려면 다음 순서로 진행합니다.

1. 목록에서 삭제할 강의실의 `삭제`를 누릅니다.
2. 삭제 확인 화면에서 연결된 예약과 반복 예약 건수를 확인합니다.
3. 확인 입력란에 강의실명을 정확히 다시 입력합니다.
4. `영구 삭제`를 누릅니다.

삭제된 강의실 row는 복구할 수 없습니다. 기존 예약과 반복 예약은 삭제하지 않고 보존용 시스템 강의실로 옮기며, 화면에는 원래 강의실명과 삭제 상태가 함께 표시됩니다. 자세한 처리 기준은 [강의실 삭제 정책](room-deletion-policy.md)을 참고하세요.

## 운영 설정 관리 방법

왼쪽 메뉴에서 `운영 설정`으로 이동합니다.

운영 설정에서 관리하는 항목은 다음과 같습니다.

- 기관명
- 기관 로고
- 공개 예약 접수 사용 여부
- 공개 안내 문구
- 접수 중지 안내 문구
- 학기 시작일과 종료일
- 운영 시작 시간과 종료 시간
- 예약 단위
- 최소/최대 예약 시간
- 예약 가능 요일
- 문의 이메일, 문의 전화번호
- 예약 완료 안내 문구

설정 변경 절차는 다음과 같습니다.

1. 현재 설정 값을 확인합니다.
2. 필요한 항목만 수정합니다.
3. 예약 가능 기간, 시간, 요일, 예약 단위가 서로 맞는지 확인합니다.
4. `설정 저장`을 누릅니다.
5. 저장 성공 메시지가 보이면 변경이 반영된 상태입니다.

운영상 중요한 점은 다음과 같습니다.

- `공개 예약 접수 사용`을 끄면 공개 예약 등록이 막힙니다. 이미 등록된 예약을 자동 취소하지는 않습니다.
- 학기 기간 밖의 날짜는 예약할 수 없습니다.
- 운영 시간 밖의 시간대는 예약할 수 없습니다.
- 예약 단위는 `5`, `10`, `15`, `30`분 중에서 선택하며 수동 예약 시간 입력의 증분으로 사용합니다.
- 시간표 눈금과 빈칸 상호작용 시작 후보는 예약 단위와 관계없이 항상 30분 간격입니다.
- 운영 시작·종료 시간은 30분 단위여야 하고, 최소·최대 예약 시간은 선택한 예약 단위로 나누어떨어져야 합니다.
- 자동 제안 길이는 `max(30분, 최소 예약 시간)`입니다. 최대 예약 시간은 이 자동 제안 길이보다 짧을 수 없습니다.
- 최소 예약 시간은 운영 시간 범위 안에 들어가야 하며 최대 예약 시간은 최소 예약 시간보다 짧을 수 없습니다.
- 공개 및 관리자 예약의 전화번호는 현재 필수 입력값입니다. 이를 켜고 끄는 운영 설정은 없습니다.

기관 로고는 PNG 또는 JPEG 정사각형 이미지만 업로드할 수 있습니다. 새 로고를 저장하면 이전 로고는 더 이상 참조되지 않을 때 정리되며, 미사용 업로드 파일은 기본적으로 매일 새벽 정리됩니다.

반복 예약 태그를 관리하려면 `운영 설정` 상단 또는 왼쪽 메뉴의 `태그 설정`으로 이동합니다. 태그 이름과 색상을 등록·수정·삭제할 수 있습니다. 사용 중인 태그를 삭제하면 기존 반복 예약의 태그 참조는 `없음`으로 변경됩니다.

## 단건 예약 등록/수정/승인/취소/삭제 방법

새 단건 예약은 왼쪽 메뉴의 `시간표`에서 등록합니다. 별도의 `예약 등록` 메뉴나 신규 등록 전용 페이지는 없습니다.

단건 예약 등록 절차는 다음과 같습니다.

1. `시간표`에서 날짜 보기 또는 강의실 주간 보기를 엽니다.
2. 빈 시간대를 누르거나 화면 상단의 `예약 신청`을 누릅니다.
3. 강의실, 시작 시간, 종료 시간을 확인합니다. 빈 시간대에서 시작하면 선택한 값이 미리 채워집니다.
4. 상태를 선택합니다. 관리자 등록 예약의 기본 상태는 `승인`입니다.
5. 신청자 이름, 이메일, 전화번호, 예약 목적을 입력합니다.
6. 필요한 경우 처리 메모를 남깁니다.
7. `예약 신청 저장`을 누릅니다.
8. 예약 목록 또는 시간표에서 생성된 예약을 열어 입력 내용과 상태를 확인합니다.

예약 수정 절차는 다음과 같습니다.

1. `예약 목록`에서 예약을 검색합니다.
2. 예약 상세 화면으로 들어갑니다.
3. `예약 수정`을 누릅니다.
4. 강의실, 신청자 정보, 목적, 시간, 상태, 메모를 수정합니다.
5. 저장 후 상세 화면에서 변경 내용을 확인합니다.

예약 승인/취소 절차는 다음과 같습니다.

1. 예약 상세 화면으로 이동합니다.
2. 처리 메모에 승인 또는 취소 사유를 적습니다.
3. 승인하려면 `승인 처리`를 누릅니다.
4. 취소하려면 `취소 처리`를 누릅니다.
5. 화면 하단 감사 이력에서 상태 변경과 메모가 남았는지 확인합니다.

예약을 영구 삭제하려면 다음 순서로 진행합니다.

1. 예약 상세 화면에서 `예약 삭제`를 누릅니다.
2. 필요한 경우 삭제 사유를 처리 메모에 입력합니다.
3. 확인 창에서 `예약 삭제`를 다시 누릅니다.
4. 이동된 감사 이력 화면에서 `DELETED` 기록과 삭제 당시 스냅샷을 확인합니다.

삭제된 예약은 상세 조회하거나 복구할 수 없습니다. 삭제 감사 이력은 예약 row가 없어도 유지됩니다.

기존 예약을 복제하려면 상세 화면에서 `복제`를 누릅니다. 시간표 예약 신청 패널에 강의실과 신청자 정보가 채워지지만 시작·종료 시간과 반복 예약 연결 정보는 복제하지 않습니다. 새 시간을 선택해 저장하면 원본과 독립된 단건 예약이 생성됩니다.

상태 의미는 다음과 같습니다.

- `REQUESTED`: 승인 대기 상태입니다.
- `CONFIRMED`: 승인된 예약입니다.
- `CANCELLED`: 취소된 예약입니다.

주의: `REQUESTED`와 `CONFIRMED`는 모두 충돌 검사 대상입니다. 같은 강의실의 시간이 겹치면 새 예약 등록 또는 수정이 실패합니다. `CANCELLED` 예약은 충돌 대상에서 제외됩니다.

공개 사용자는 과거 시간의 예약 생성·수정·availability 확인을 할 수 없습니다. 관리자는 운영 정책과 충돌 검증을 만족하는 범위에서 과거 단건 예약 생성·수정과 반복 예약을 할 수 있습니다.

예약 단위를 변경해도 기존 예약과 반복 예약 시간은 자동 변환하거나 소급 검증하지 않습니다. 기존 예약의 승인·취소·삭제는 유지되지만, 시간이나 내용을 수정해 다시 저장할 때는 현재 예약 단위를 적용합니다.

상태와 감사 이력 action 용어는 서로 구분해서 해석합니다. `REQUESTED`, `CONFIRMED`, `CANCELLED`는 예약의 현재 상태입니다. `APPROVED`는 예약 상태가 아니라, 관리자가 승인 처리를 수행했다는 감사 이력 action입니다. 따라서 승인 처리 후 예약 상태는 `CONFIRMED`로 저장되고, 감사 이력에는 `APPROVED` action이 기록됩니다.

## 반복 예약 미리보기/등록/취소 방법

왼쪽 메뉴에서 `반복 예약`으로 이동합니다.

반복 예약은 반복 규칙을 계속 실행하는 기능이 아니라, 미리보기 결과를 바탕으로 여러 일반 예약을 한 번에 등록하는 기능입니다. 등록된 각 회차는 일반 예약 상세에서 개별 수정 또는 개별 취소할 수 있고, 반복 예약 화면은 같은 묶음의 회차를 보여줍니다.

반복 예약 미리보기 절차는 다음과 같습니다.

1. 강의실을 선택합니다.
2. 신청자 이름, 이메일, 전화번호, 예약 목적을 입력합니다.
3. 시작일과 종료일을 입력합니다.
4. 시작 시간과 종료 시간을 입력합니다.
5. 반복 요일을 선택합니다.
6. 등록 정책을 선택합니다.
7. `미리보기`를 누릅니다.
8. 전체 후보 수, 가능 수, 충돌 수를 확인합니다.
9. 미리보기 표에서 날짜별 가능/충돌 결과와 사유를 확인합니다.

반복 예약 등록 절차는 다음과 같습니다.

1. 미리보기 결과를 확인합니다.
2. 등록 정책이 의도와 맞는지 다시 확인합니다.
3. `반복 예약 등록`을 누릅니다.
4. 등록 완료 결과의 등록 건수, 건너뛴 건수, 실패 건수를 확인합니다.
5. 반복 예약 목록 또는 예약 목록에서 실제 등록된 예약을 확인합니다.

반복 예약 등록 정책은 다음과 같이 해석합니다.

- `FAIL_ALL`: 충돌 또는 정책 위반이 하나라도 있으면 전체 등록을 중단합니다.
- `SKIP_CONFLICTS`: 충돌 또는 정책 위반 건은 건너뛰고 가능한 건만 등록합니다.

반복 예약 취소 절차는 다음과 같습니다.

1. 반복 예약 목록에서 취소할 항목의 `취소`를 누르거나 상세 화면으로 들어갑니다.
2. 취소 메모를 입력합니다.
3. `반복 예약 취소`를 누릅니다.
4. 반복 예약 상태가 취소로 바뀌었는지 확인합니다.
5. 연결된 활성 예약이 `CANCELLED`로 바뀌었는지 예약 목록에서 확인합니다.

주의: 반복 예약 취소는 반복 예약 묶음 정보를 소프트 삭제하고, 연결된 활성 예약을 일괄 취소합니다. 이미 개별 취소된 예약은 다시 취소할 필요가 없습니다.

## 충돌이 발생했을 때 해석 방법

충돌은 같은 강의실에서 예약 시간이 겹치는 경우입니다. 시스템은 `REQUESTED`, `CONFIRMED` 상태 예약을 모두 점유 중인 시간으로 봅니다.

자주 보는 충돌/정책 사유는 다음과 같습니다.

- `TIME_SLOT_CONFLICT`: 같은 강의실의 기존 예약과 시간이 겹칩니다.
- `ROOM_DISABLED`: 해당 강의실이 예약 가능 상태가 아닙니다.
- `RESERVATION_DISABLED`: 공개 예약 접수가 중지되어 있습니다.
- `OUTSIDE_SEMESTER_PERIOD`: 운영 설정의 학기 기간 밖입니다.
- `OUTSIDE_OPERATING_DAYS`: 예약 가능 요일이 아닙니다.
- `OUTSIDE_OPERATING_HOURS`: 운영 시간 밖이거나 하루를 넘기는 예약입니다.
- `INVALID_DURATION`: 최소/최대 예약 시간 범위를 벗어났습니다.
- `INVALID_SLOT_UNIT`: 설정된 예약 단위에 맞지 않는 시간입니다.
- `VALIDATION_ERROR`: 필수 입력 누락, 시작/종료 시간 오류 등 입력값 문제입니다.

충돌이 나면 먼저 다음 순서로 확인합니다.

1. 같은 강의실의 같은 날짜 예약 목록을 확인합니다.
2. 취소되지 않은 예약과 시간이 겹치는지 확인합니다.
3. 운영 설정의 기간, 요일, 시간, 예약 단위와 맞는지 확인합니다.
4. 반복 예약이라면 미리보기에서 충돌 날짜만 건너뛸지, 전체 등록을 중단할지 결정합니다.

## 대표 오류 메시지

관리자 화면은 백엔드의 영문 오류를 그대로 보여주지 않고 관리자가 이해하기 쉬운 문구로 바꿔 표시합니다.

- `로그인이 필요합니다. 다시 로그인해 주세요.`: 세션이 없거나 만료된 상태입니다.
- `같은 강의실의 같은 시간대에 이미 예약이 있습니다.`: 기존 예약과 시간이 겹칩니다.
- `운영 시간 안에서만 예약할 수 있습니다.`: 운영 설정의 시작/종료 시간을 벗어났습니다.
- `예약 가능한 요일이 아닙니다.`: 운영 설정의 예약 가능 요일과 맞지 않습니다.
- `다른 사용자가 먼저 수정했습니다.`: 운영 설정 등에서 동시 수정이 발생했습니다. 화면을 새로고침한 뒤 다시 시도합니다.
- `요청한 대상을 찾을 수 없습니다.`: 이미 삭제되었거나 다른 화면에서 변경된 강의실/예약/반복 예약일 수 있습니다.
- `서버에 연결할 수 없습니다.`: 네트워크 또는 서버 상태를 확인한 뒤 다시 시도합니다.
- `요청이 너무 많습니다. 잠시 후 다시 시도해 주세요.`: 비로그인 또는 공개 요청이 IP별 요청 제한을 초과했습니다. 응답의 재시도 시간을 기다린 뒤 다시 시도합니다.

## CSV 내보내기 사용 방법

예약 목록 화면에서 현재 검색 조건으로 CSV를 내보낼 수 있습니다.

사용 절차는 다음과 같습니다.

1. `예약 목록`으로 이동합니다.
2. 상태, 강의실, 시작일, 종료일, 검색어 조건을 설정합니다.
3. 조회 결과가 의도한 범위인지 확인합니다.
4. `CSV 내보내기`를 누릅니다.
5. `reservations.csv` 파일을 받아 엽니다.

CSV에는 예약 ID, 강의실명, 신청자명, 이메일, 전화번호, 목적, 시작/종료 시각, 상태, 등록 경로, 반복 예약 ID, 등록 시각이 포함됩니다. 시간은 한국 시간 기준 `yyyy-MM-dd HH:mm:ss` 형식으로 내려갑니다. Excel 호환을 위해 UTF-8 BOM이 포함됩니다.

주의: 왼쪽 메뉴의 `CSV 내보내기` 링크는 별도 필터 없이 전체 예약 CSV를 요청합니다. 특정 조건만 내려받으려면 예약 목록 화면에서 조건을 적용한 뒤 `CSV 내보내기`를 사용하세요.

## 감사 이력 확인 방법

예약별 감사 이력은 예약 상세 화면 하단에서 확인합니다.

전체 감사 이력은 왼쪽 메뉴의 `감사 이력`에서 확인합니다.

감사 이력 검색 절차는 다음과 같습니다.

1. `감사 이력`으로 이동합니다.
2. 필요한 경우 예약 ID, 강의실, 처리 유형, 시작일, 종료일을 입력합니다.
3. `조회`를 누릅니다.
4. 처리 시각, 처리 유형, 상태 변경, 처리자, 연결 예약, 메모를 확인합니다.
5. 특정 예약을 자세히 보려면 `상세 보기`를 누릅니다.

주요 처리 유형은 다음과 같습니다.

- `CREATED`: 공개 예약 등록
- `CREATED_BY_ADMIN`: 관리자 예약 등록
- `RECURRENCE_GENERATED`: 반복 예약 등록
- `UPDATED`: 예약 수정
- `APPROVED`: 예약 승인 감사 이력. 예약 상태 값이 아니라 감사 이력 action입니다.
- `CANCELLED`: 예약 취소
- `DELETED`: 예약 영구 삭제. 삭제 당시 예약 정보는 감사 이력 스냅샷으로 남습니다.
- `RECURRENCE_CANCELLED`: 반복 예약 취소로 연결 예약 취소

주의: 현재 감사 이력은 예약 감사 이력 중심입니다. 강의실 변경, 운영 설정 변경, 로그인 이력은 별도 감사 화면으로 제공되지 않습니다.

## 운영 시 주의사항

- 운영 시간, 학기 기간, 예약 가능 요일을 바꾸면 이후 등록/수정되는 예약에 정책이 적용됩니다.
- 이미 존재하는 예약이 운영 설정 변경으로 자동 취소되지는 않습니다.
- 같은 강의실 같은 시간대에는 승인 대기와 승인 예약이 동시에 존재할 수 없습니다.
- 반복 예약을 등록하기 전에는 반드시 미리보기 충돌 수를 확인합니다.
- CSV에는 신청자 개인정보가 포함될 수 있으므로 공유 범위를 제한합니다.
- local/dev/prod 관리자 계정은 환경변수로 주입하며 test/E2E의 기본 계정을 운영 환경에서 사용하지 않습니다.
- 관리자가 화면에서 관리자 계정을 추가하거나 비밀번호를 변경할 수 없습니다.
- 공개 및 비로그인 API 요청은 IP별 빈도 제한을 받습니다. 인증된 관리자 요청은 이 제한을 건너뜁니다.
- 장애 조사 시 예약 상세 이력과 감사 이력을 먼저 확인합니다.

## 아직 미구현 또는 관리자가 할 수 없는 것

- 관리자 계정 등록, 비밀번호 변경, 역할 관리
- 삭제한 강의실 또는 예약 복구
- 반복 예약 수정 화면
- 반복 예약에 연결된 예약만 모아보는 전용 필터
- 공개 화면의 배포·사용자 지원·장애 대응 운영 절차
- 이메일, 문자, 카카오톡 등 알림 발송
- 예약 승인/취소 알림 자동 발송
- 첨부파일, 장비 대여, 승인 라인, 다중 관리자 결재
- 강의실별 운영 시간 또는 예외 휴일 설정
- 감사 이력 CSV 내보내기
- 운영 설정 변경 이력 확인
- 개인정보 마스킹, 보관 기간, 파기 정책 자동화
- 관리자 화면에서 DB 백업/복구 실행
